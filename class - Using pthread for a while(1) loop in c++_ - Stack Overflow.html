If you are really restricted to pthreads the following might help.

#include <pthread.h> 
#include <iostream> 
#include <vector> 
#include <algorithm> 
#include <unistd.h> 

class test { 
public: 
    void save(...) {}; 
    void test1() { 
        double i = 1; 

        while(true) 
        { 
            save(i, 0, 0, 0, 0, 0); 
            i++; 

            std::cerr << " ** " << std::endl; 
            usleep( 1000000 ); 

        } 
    } 

    static void * run(void * arg ) { 
        ((test *)arg)->test1();
        return NULL; 
    } 
}; 


int main() 
{ 
    pthread_t thr; 
    test tst; 
    pthread_create(&thr, NULL,test::run,&tst); 
    pthread_join(thr,NULL); 
    return 1; 
}

The most important part for a running pthread is a pointer to a C-Style function which take a void * argument. Non static C++ member functions always take this as a hidden argument so they are no direct candidate for a pthread. I always create a static run(...) method which explicitly takes the pointer to the object and forwards the call to the non-static member function.

If you are allowed to use more modern C++ features I would follow @Ulrich Eckhardt and use the std::thread construct.

PS: Added missing retun statement in run(...)
